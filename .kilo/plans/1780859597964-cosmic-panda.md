# Plan: World-Class SaaS Project Management Dashboard

## Current State Analysis

The project already has strong foundations:
- **Tech stack**: React 19 + TypeScript + Tailwind CSS v4 + Framer Motion + Vite
- **Existing App.tsx**: Has sidebar, top navbar, welcome section, analytics cards, project progress, and team activity
- **Existing Background.tsx**: Full-screen animated layered radial gradients with particles, light streaks, grid overlay, and moving blobs
- **Existing GlassCard.tsx**: Glassmorphism card component with hover effects
- **Missing active integration**: `main.tsx` does not use `BrowserRouter`, so navigation links and page routes are non-functional
- **Missing widgets in standalone App.tsx**: Calendar Widget, Kanban Preview, Time Tracking, Reports Widget

---

## Step-by-Step Implementation Plan

### 1. Fix and Modernize Auth Context
- **File**: `dashboard/src/context/AuthContext.jsx`
- **Action**: Add `setAuthView` to `useMemo` dependency array (state setter should be included per lint rules)
- **Action**: Ensure it exports clean types and works with TSX components

### 2. Enable Routing in Application Root
- **File**: `dashboard/src/main.tsx`
- **Action**: Wrap `<App />` with `BrowserRouter` from `react-router-dom` so navigation works
- **Action**: Create a Layout route that renders `DashboardLayout` with an `<Outlet />`

### 3. Create Central Router Configuration
- **New File**: `dashboard/src/App.tsx`
- **Action**: Replace current App.tsx with `createBrowserRouter` + `RouterProvider`
- **Action**: Define routes: `/` (Dashboard), `/projects`, `/tasks`, `/kanban`, `/calendar`, `/team`, `/files`, `/time`, `/reports`, `/settings`
- **Action**: For pages without dedicated components, render `Placeholder.tsx`

### 4. Enhance Dashboard Page with All Missing Widgets
- **File**: `dashboard/src/pages/Dashboard.tsx`
- **Action**: Import and integrate existing components:
  - `CalendarWidget` (already exists at `components/CalendarWidget.tsx`)
  - `KanbanPreview` (already exists at `components/KanbanPreview.tsx`)
  - `TimeTracking` (already exists at `components/TimeTracking.tsx`)
  - `ReportsWidget` (already exists at `components/ReportsWidget.tsx`)
- **Action**: Keep existing Welcome Hero, StatCards, ProjectProgress, TeamActivity
- **Action**: Add responsive grid:
  - Stat cards: `grid-cols-2 sm:grid-cols-3 xl:grid-cols-5`
  - Kanban Preview: full width
  - ProjectProgress + TeamActivity: 2-column grid on lg
  - Calendar + TimeTracking + Reports: 3-column grid on lg

### 5. Polish Design (Premium "Worth Millions" Feel)
- **Files**: `index.css`, `components/GlassCard.tsx`, components
- **Actions**:
  - Add `Space Grotesk` font family for headings (already imported)
  - Ensure all cards use `backdrop-blur-3xl` and `rounded-[24px]`
  - Add neon blue/cyan glow on hover for all interactive elements
  - Ensure smooth transitions (`transition-all duration-300/500`)
  - Add `framer-motion` entrance animations to every widget section
  - Add subtle floating animation to stat cards (using `animate-bounce-subtle` or custom Framer Motion)
  - Add micro-interactions: scale on hover, glow intensification

### 6. Responsive Behavior
- **All layout files**
- **Actions**:
  - Mobile: Collapsible sidebar (already implemented in Sidebar.tsx with mobile drawer)
  - Tablet: 2-column grids where desktop is 3-4 columns
  - Desktop: Full 3-4 column layout with max-width container
  - Ensure text scales down on mobile (`text-2xl md:text-4xl`)

### 7. Unify Imports and Fix Type Safety
- **Files**: Any `.tsx` importing `.jsx` files
- **Actions**:
  - Update imports in `Dashboard.tsx` and `DashboardLayout.tsx` to use correct file extensions
  - Ensure `AuthContext.jsx` types are compatible (use `React.ReactNode` in JSX since it's a JS file)

### 8. Build and Verify
- **Commands**:
  - `npm run setup` (installs root + dashboard deps)
  - `npm run dev:app` (starts Vite dev server)
  - `npm run build:app` (production build)
  - `npm start` (serves built app on port 3000)
